# 🧠 Feedforward Neural Network Engine — From Scratch

> A fully functional neural network built with **NumPy only** — no PyTorch, no TensorFlow, no autograd. Every component derived and implemented by hand: forward propagation, backpropagation, He initialization, SGD, and Adam. Trained on MNIST and benchmarked against sklearn and Keras to validate correctness.

## 🏗️ Architecture

```
Input Layer      →   784 neurons  (28×28 flattened pixels, normalized to [0,1])
Hidden Layer 1   →   128 neurons  (ReLU activation)
Hidden Layer 2   →    64 neurons  (ReLU activation)
Output Layer     →    10 neurons  (Softmax activation, one per digit class)
```

**Design choices explained:**

- **He (Kaiming) initialization** — Weights drawn from `N(0, sqrt(2/n_prev))`. Critical for ReLU networks; standard normal init causes variance to explode or vanish across layers, breaking gradient flow entirely.
- **ReLU hidden activations** — Avoids the vanishing gradient problem that plagued sigmoid/tanh in deep networks. Simple, fast, and effective.
- **Softmax output + cross-entropy loss** — The gradient of this combination simplifies to `AL - Y_onehot`, which is one of the cleaner results in the backprop derivation.
- **Full-batch gradient descent** — All 41,000 training samples per step. Produces exact (low-variance) gradients at the cost of slower wall-clock time vs mini-batch. No shuffling required.

## ⚙️ What's Implemented from Scratch

| Component | Implementation |
|---|---|
| 🔁 **Forward propagation** | Fully vectorized — `Z = W @ A_prev + b`, column-major layout (features × samples) |
| 🔙 **Backpropagation** | Exact chain-rule through Softmax+CrossEntropy → ReLU → linear layers |
| 📉 **SGD** | `W -= lr * dW` — baseline optimizer to compare against Adam |
| ⚡ **Adam optimizer** | First moment `v`, second moment `s`, bias correction, per-parameter adaptive rates |
| 🎲 **He initialization** | Layer-wise scaled init: `W = randn(n_curr, n_prev) * sqrt(2/n_prev)` |
| 📦 **Model serialization** | `pickle`-based save/load of weights dict + architecture list |
| 📈 **Training history** | Loss, train accuracy, val accuracy logged every 50 epochs |
| 🎯 **Prediction** | `argmax(softmax(forward(X)))` for class labels; `forward(X)` for probabilities |

## 🔬 Adam Optimizer — Under the Hood

The Adam implementation uses the full update rule with bias correction:

```python
# Moment estimates
v[dW] = β₁ * v[dW] + (1 - β₁) * dW       # first moment  (mean)
s[dW] = β₂ * s[dW] + (1 - β₂) * dW²      # second moment (uncentered variance)

# Bias correction (critical in early training when moments are near zero)
v_hat = v[dW] / (1 - β₁ᵗ)
s_hat = s[dW] / (1 - β₂ᵗ)

# Parameter update
W -= lr * v_hat / (sqrt(s_hat) + ε)
```

Hyperparameters used: `β₁ = 0.9`, `β₂ = 0.999`, `ε = 1e-8`, `lr = 0.001` — the defaults from the original paper. This is why Adam converged 3.2% higher than SGD on the same architecture.

## 🚀 Running Locally

**1. Clone and install**
```bash
git clone https://github.com/AbdullahPatti/Feedforward-Neural-Network-Engine-From-Scratch.git
cd Feedforward-Neural-Network-Engine-From-Scratch
pip install -r requirements.txt
```

**2. Get the MNIST dataset**

Download `train.csv` from [Kaggle Digit Recognizer](https://www.kaggle.com/competitions/digit-recognizer/data) and place it in the repo root. The notebook expects it at `./train.csv`.

**3. Train the models**

Open `Neural_Networks.ipynb` → Kernel → **Restart & Run All**. This runs all four models and exports `results.json`. Training takes ~10 minutes total on a standard CPU.

**4. Launch the dashboard**
```bash
streamlit run app.py
```

## 💡 Why Build It from Scratch

Most ML courses hand you a framework and have you call `.fit()`. That works for building things, but it doesn't build understanding. When you implement backprop yourself — computing `dZ = AL - Y_onehot` for the output layer, propagating gradients through ReLU, handling the Jacobian of softmax — the math stops being abstract. You see exactly why He init matters, why Adam beats SGD here, why full-batch training is slower but more stable. That understanding carries forward into every framework you use after.
